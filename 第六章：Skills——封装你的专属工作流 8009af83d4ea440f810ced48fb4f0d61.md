# Skills——封装你的专属工作流 🟡

> **核心价值**：Skills 把反复使用的工作流封装为可复用的模板，一次定义，多次调用。
> 

---

## 6.1 什么是 Skills？

Skills 是存放在 `.claude/commands/` 目录下的 Markdown 文件，每个文件就是一个可调用的「技能」。

**Skills vs [CLAUDE.md](http://CLAUDE.md) 的分工：**

- [**CLAUDE.md**](http://CLAUDE.md) = 项目上下文和规范（情景记忆）
- **Skills** = 可复用的工作流模板（动作模板）

**调用方式：**

```bash
# 在对话中直接使用斜杠命令
/日报
/部署检查
/代码审查
```

---

## 6.2 Custom Commands 自定义斜杠命令

### 创建你的第一个命令

```bash
# 创建命令目录
mkdir -p .claude/commands
```

**创建日报命令：**

```bash
cat > .claude/commands/日报.md << 'EOF'
# 生成工作日报

扫描今天修改过的所有文件（git diff --stat），
结合 .claude/worklog.md 中今天的记录，
按以下格式生成日报：

## 今日完成
- （从文件变更中提取）

## 遇到的问题
- （从注释和提交信息中提取）

## 明日计划
- （根据未完成任务推断）

输出保存到 daily-reports/YYYY-MM-DD.md
EOF
```

使用时：在对话中输入 `/日报` 即可。

### 更多命令示例

**部署前检查：**

```markdown
# 部署前检查

执行部署前的安全检查：

1. 检查是否有硬编码密钥（grep -r "password\|secret\|api_key" src/）
2. 检查所有测试是否通过（pytest tests/ -v）
3. 检查依赖是否有已知漏洞（safety check）
4. 检查数据库迁移文件是否就绪

发现问题输出具体报告，没有问题输出 "READY TO DEPLOY"
```

**代码审查：**

```markdown
# 代码审查

对当前 Git diff 进行代码审查，关注：

## 安全性
- 是否有 SQL 注入风险
- 是否有硬编码的敏感信息
- 是否有未验证的用户输入

## 代码质量
- 是否有重复代码可以提取
- 是否有明显的性能问题
- 是否符合项目的命名规范

## 建议
- 列出所有建议，标明优先级（必改/建议改/可选）

输出 Markdown 格式的审查报告
```

## 6.3 目录结构与跨项目共享

```
项目根目录/
└── .claude/
    ├── commands/             ← 项目专属斜杠命令
    │   ├── 日报.md
    │   ├── 部署检查.md
    │   └── 代码审查.md
    └── CLAUDE.md

~/.claude/
└── commands/             ← 全局命令（所有项目可用）
    ├── 翻试.md           ← 随时可用的全局命令
    └── 总结.md
```

**跨目录自动加载（2026年新特性）：**

```bash
# --add-dir 指定的目录中的 Skills 会自动加载
claude --add-dir ~/shared-skills

# 适合团队共享的通用工作流
claude --add-dir ~/公司共用skills/
```

## 6.5 Plugins 插件

**插件 vs Skills 的区别：**

- Skills = 你写的 Markdown 工作流模板
- Plugins = 官方生态的外部连接器（包含实际的代码逻辑）

**当前插件市场（38+官方连接器）：**

| 类别 | 插件 |
| --- | --- |
| 文件存储 | Google Drive, Box, Dropbox |
| 邮件通信 | Gmail, Outlook, Notion Mail |
| 项目管理 | Jira, Linear, Asana, Notion |
| 山云协作 | Slack, Microsoft Teams |
| 文件签名 | DocuSign |
| 金融数据 | FactSet, Bloomberg |
| 日历 | Google Calendar, Notion Calendar |

**安装插件：**

```bash
# 通过 npm 安装插件
npm install -g @claude-plugins/google-drive
npm install -g @claude-plugins/slack
npm install -g @claude-plugins/notion

# 在对话中查看已安装插件
/mcp
```

## 🔬 实战案例

### 案例1：创建「日报生成器」命令（完整流程）

```bash
# 第一步：创建命令目录
mkdir -p .claude/commands

# 第二步：写入命令文件
cat > .claude/commands/日报.md << 'SKILL'
# 生成工作日报

执行以下步骤生成今日工作日报：

1. 运行 `git diff --stat HEAD~1` 获取今日文件变更
2. 运行 `git log --oneline --since="today" ` 获取今日提交记录
3. 读取 .claude/worklog.md 中今天的手动备注（如果存在）

整合以上信息，按下面符式生成日报：

---
【YYYY-MM-DD 工作日报】

**今日完成**
- ...

**遇到的问题**
- ...

**明日计划**
- ...

---

将日报保存到 daily-reports/YYYY-MM-DD.md，其中 YYYY-MM-DD 替换为实际日期。
SKILL

# 第三步：测试使用
# 在对话中输入：
claude
# 然后输入：
/日报
```

### 案例2：数据清洗标准化 Skill

```markdown
# 数据清洗标准化

对当前目录下所有 CSV 文件执行标准化清洗：

1. 模板列名统一处理（参考 data/column-mapping.json）
2. 日期格式统一为 YYYY-MM-DD
3. 空值处理：数值型填 0，文字型填 "未知"
4. 去除完全重复的行
5. 生成清洗报告（处理了多少行、修正了多少个问题）

将清洗后的文件保存为原文件名_cleaned.csv
```

### 案例3：安装并使用 10 个社区 Skills

```bash
# 一键安装常用 Skills
mkdir -p .claude/commands

# 转到项目目录
cd 你的项目目录

# 安装常用 Skills
for skill in sequential-thinking code-review git-commit doc-writer test-writer; do
  curl -o .claude/commands/${skill}.md \
    "https://raw.githubusercontent.com/VoltAgent/awesome-agent-skills/main/skills/${skill}.md"
done

echo "安装完成！可用的 Skills："
ls .claude/commands/
```
