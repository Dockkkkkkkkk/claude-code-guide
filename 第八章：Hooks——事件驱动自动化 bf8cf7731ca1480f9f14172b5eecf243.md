# Hooks——事件驱动自动化 🔴

> **核心价值**：Hooks 让你在 Claude Code 执行操作的**前后**自动运行自定义脚本，建立全自动化工作流。
> 

---

## 8.1 什么是 Hooks？

**Hooks** 是 Claude Code 在执行特定操作时自动触发的外部命令。

**Hooks vs Skills vs MCP 的分工：**

| 机制 | 处理的是 | 触发旹式 |
| --- | --- | --- |
| [**CLAUDE.md**](http://CLAUDE.md) | 情景/背景知识 | 每次启动自动加载 |
| **Skills** | 可复用任务模板 | 手动调用 |
| **MCP** | 外部工具连接 | Claude 主动调用 |
| **Hooks** | 操作过程中的事件 | **自动触发** |

2026年初 Hooks 正式落地，完整文档已发布。

---

## 8.2 五种 Hook 类型详解

| **Hook 类型** | **触发时机** | **典型用途** |
| --- | --- | --- |
| `PreToolUse` | 工具调用前 | 权限检查、参数校验、审批流程 |
| `PostToolUse` | 工具调用后 | 自动格式化、日志记录、测试运行 |
| `Notification` | Claude 发通知时 | 转发到钉钉/企业微信/Slack |
| `Stop` | 会话结束时 | 生成会话摘要、清理临时文件 |
| `SubagentStop` | 子 Agent 完成时 | 汇总子任务结果 |

---

## 8.3 Hooks 配置方式

在 `~/.claude/settings.json` 中配置：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write \"$CLAUDE_TOOL_INPUT_PATH\""
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npm test -- --testPathPattern=$CLAUDE_TOOL_INPUT_PATH 2>&1 | tail -5"
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/scripts/notify-done.sh"
          }
        ]
      }
    ]
  }
}
```

### Hook 中可用的环境变量

| 变量 | 说明 |
| --- | --- |
| `CLAUDE_TOOL_NAME` | 被调用的工具名称 |
| `CLAUDE_TOOL_INPUT_PATH` | 文件路径（Write/Edit时） |
| `CLAUDE_TOOL_INPUT` | 工具输入参数（JSON） |
| `CLAUDE_TOOL_OUTPUT` | 工具执行结果 |
| `CLAUDE_SESSION_ID` | 当前会话 ID |

## 8.4 实用 Hook 场景全解

### 场景1：文件保存后自动 prettier 格式化

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "cd $(dirname $CLAUDE_TOOL_INPUT_PATH) && prettier --write $CLAUDE_TOOL_INPUT_PATH 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

### 场景2：代码修改后自动运行测试

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "if echo $CLAUDE_TOOL_INPUT_PATH | grep -q '\.test\.'; then echo '跳过测试文件本身'; else npm test -- --bail --testPathPattern=$(basename $CLAUDE_TOOL_INPUT_PATH .js) 2>&1 | tail -10; fi"
          }
        ]
      }
    ]
  }
}
```

### 场景3：任务完成实时发送企业微信通知

创建 `~/.claude/scripts/notify-wechat.sh`：

```bash
#!/bin/bash
# 任务完成通知脚本

WECHAT_WEBHOOK="你的企业微信机器人_webhook_url"

curl -s -X POST "$WECHAT_WEBHOOK" \
  -H 'Content-Type: application/json' \
  -d '{
    "msgtype": "text",
    "text": {
      "content": "Claude Code 任务完成 \u2705\n会话 ID: '"$CLAUDE_SESSION_ID"'"
    }
  }'
```

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/scripts/notify-wechat.sh"
          }
        ]
      }
    ]
  }
}
```

### 场景4：全链路操作审计日志

```bash
# 创建日志脚本
cat > ~/.claude/scripts/audit-log.sh << 'EOF'
#!/bin/bash
LOG_FILE="$HOME/.claude/audit.log"
echo "[$(date '+%Y-%m-%d %H:%M:%S')] Tool=$CLAUDE_TOOL_NAME File=$CLAUDE_TOOL_INPUT_PATH Session=$CLAUDE_SESSION_ID" >> $LOG_FILE
EOF

chmod +x ~/.claude/scripts/audit-log.sh
```

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude/scripts/audit-log.sh"
          }
        ]
      }
    ]
  }
}
```

### 场景5：高危操作自动要求二次确认

```bash
# 创建确认脚本
cat > ~/.claude/scripts/confirm-danger.sh << 'EOF'
#!/bin/bash
# 如果命令包含高危操作，要求确认
if echo "$CLAUDE_TOOL_INPUT" | grep -qE 'rm -rf|DROP TABLE|DELETE FROM.*WHERE|kubectl delete'; then
  echo -n "❗ 警告：检测到高危命令，确认执行？(y/N): " >&2
  read -r response
  if [ "$response" != "y" ] && [ "$response" != "Y" ]; then
    echo "BLOCK" # 返回 BLOCK 将默认阻止执行
    exit 1
  fi
fi
EOF

chmod +x ~/.claude/scripts/confirm-danger.sh
```

## 🔬 实战案例

### 案例1：文件保存后自动格式化

```bash
# 安装 prettier
npm install -g prettier

# 配置 Hook
cat >> ~/.claude/settings.json << 'EOF'
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "prettier --write \"$CLAUDE_TOOL_INPUT_PATH\" 2>/dev/null || true"
      }]
    }]
  }
}
EOF

# 现在每次 Claude Code 修改 JS/TS 文件，
# 就会自动运行 prettier 格式化——无需手动操作
```

### 案例2：任务完成自动发送企业微信通知

```bash
mkdir -p ~/.claude/scripts

cat > ~/.claude/scripts/notify-done.sh << 'EOF'
#!/bin/bash
curl -s -X POST "$WECHAT_WEBHOOK" \
  -H 'Content-Type: application/json' \
  -d "{\"msgtype\":\"text\",\"text\":{\"content\":\"Claude Code 任务完成 \u2705\"}}"
EOF
chmod +x ~/.claude/scripts/notify-done.sh

export WECHAT_WEBHOOK="你的webhook地址"
```

### 案例3：建立完整操作审计日志系统

```bash
mkdir -p ~/.claude/scripts

cat > ~/.claude/scripts/full-audit.sh << 'EOF'
#!/bin/bash
LOG="$HOME/.claude/audit-$(date +%Y%m).log"
printf '[%s] %-20s %-10s %s\n' \
  "$(date '+%Y-%m-%d %H:%M:%S')" \
  "$CLAUDE_TOOL_NAME" \
  "$CLAUDE_SESSION_ID" \
  "$CLAUDE_TOOL_INPUT_PATH" >> "$LOG"
EOF
chmod +x ~/.claude/scripts/full-audit.sh

# 查看审计日志
cat ~/.claude/audit-2026-03.log
```
