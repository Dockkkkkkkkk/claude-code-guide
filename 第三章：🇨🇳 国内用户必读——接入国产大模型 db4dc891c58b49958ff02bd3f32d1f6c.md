## 3.1 为什么要换模型？

国内用户直接使用 Anthropic 官方服务面临以下问题：

- 🚫 **封号风险高**：Anthropic 对国内 IP 限制严格，账号可能随时被封
- 💳 **支付门槛**：官方订阅需要境外信用卡，获取麻烦
- 🌐 **访问不稳定**：需要持续的代理，体验差
- 💰 **费用较高**：最低 $20/月，Pro 计划 Claude Code 还有用量限制

**替换后的优势：**

- ✅ 无需代理，直接访问国内 API
- ✅ 按量付费，日常使用成本极低（¥0.5～2元/小时）
- ✅ 2026年国产顶尖模型已全面追平 Claude Sonnet 4.6
- ✅ 部分模型完全免费（GLM-4.7-Flash、GLM-Z1-Flash）

---

## 3.2 替换原理

> 修改两个核心环境变量，把请求从 Anthropic 服务器转发到支持 Anthropic API 格式的国产模型接口。
> 

Claude Code 读取以下环境变量来决定调用哪个 API：

| **变量名** | **作用** | **必填** |
| --- | --- | --- |
| `ANTHROPIC_BASE_URL` | API 请求转发地址 | ✅ 最关键 |
| `ANTHROPIC_AUTH_TOKEN` | 国产模型的 API Key | ✅ 必填 |
| `ANTHROPIC_MODEL` | 主力模型名 | 推荐填写 |
| `ANTHROPIC_SMALL_FAST_MODEL` | 快速/小任务模型 | 推荐填写 |
| `ANTHROPIC_DEFAULT_OPUS_MODEL` | 复杂任务模型（对应 Opus 角色） | 推荐填写 |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | 日常模型（对应 Sonnet 角色） | 推荐填写 |
| `ANTHROPIC_DEFAULT_HAIKU_MODEL` | 轻量模型（对应 Haiku 角色） | 推荐填写 |
| `CLAUDE_CODE_SUBAGENT_MODEL` | 子 Agent 使用的模型 | 推荐填写 |
| `API_TIMEOUT_MS` | 超时时间（推荐600000，即10分钟） | 强烈推荐 |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` | 禁用非必要流量（推荐设为1） | 强烈推荐 |

### 🖥️ 本地模型——完全离线，数据不出本机

**Ollama（推荐，最简单）**

```bash
# macOS/Linux 安装 Ollama
curl -fsSL https://ollama.ai/install.sh | sh

# Windows：下载 ollama.exe 安装包：https://ollama.ai/download

# 拉取模型
ollama pull qwen3.5:14b

# 启动服务
ollama serve

# 配置 Claude Code
export ANTHROPIC_BASE_URL=http://localhost:11434/api
export ANTHROPIC_AUTH_TOKEN=ollama
export ANTHROPIC_MODEL=qwen3.5:14b
```

**LM Studio（图形化界面）**

```bash
# 在 LM Studio 中下载并启动本地服务（端口默认1234）
export ANTHROPIC_BASE_URL=http://localhost:1234
export ANTHROPIC_AUTH_TOKEN=lmstudio
claude --model openai/你的模型名
```

## 3.4 配置持久化（永久生效）

```json

    "ANTHROPIC_AUTH_TOKEN": "sk-你的API_Key",
    "ANTHROPIC_MODEL": "deepseek-chat",
    "ANTHROPIC_SMALL_FAST_MODEL": "deepseek-chat",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "deepseek-reasoner",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "deepseek-chat",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "deepseek-chat",
    "CLAUDE_CODE_SUBAGENT_MODEL": "deepseek-chat",
    "API_TIMEOUT_MS": "600000",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  }

```

### macOS / Linux（写入 ~/.zshrc）

```bash
# 在 ~/.zshrc 末尾添加
export DEEPSEEK_API_KEY="sk-你的Key"
```

## 3.5 多模型快速切换方案

### 方案一：Shell 函数一键切换（推荐）

```bash
# 加入 ~/.zshrc 或 ~/.bashrc
function use-deepseek() {
  export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
  export ANTHROPIC_AUTH_TOKEN=$DEEPSEEK_API_KEY
  export ANTHROPIC_MODEL=deepseek-chat
  export ANTHROPIC_DEFAULT_OPUS_MODEL=deepseek-reasoner
  echo "已切换到 DeepSeek V3.2（1M上下文）"
}

function use-kimi() {
  export ANTHROPIC_BASE_URL=https://api.moonshot.cn/anthropic
  export ANTHROPIC_AUTH_TOKEN=$KIMI_API_KEY
  export ANTHROPIC_MODEL=kimi-k2.5
  echo "已切换到 Kimi K2.5（开源全球第一）"
}

function use-qwen() {
  export ANTHROPIC_BASE_URL=https://dashscope.aliyuncs.com/apps/anthropic
  export ANTHROPIC_AUTH_TOKEN=$QWEN_API_KEY
  export ANTHROPIC_MODEL=qwen3.5-plus
  echo "已切换到 Qwen3.5-Plus（原生多模态）"
}

function use-glm() {
  export ANTHROPIC_BASE_URL=https://open.bigmodel.cn/api/anthropic
  export ANTHROPIC_AUTH_TOKEN=$GLM_API_KEY
  export ANTHROPIC_MODEL=glm-5.1
  echo "已切换到 GLM-5.1（今日最新，逼近Opus 4.6）"
}

function use-minimax() {
  export ANTHROPIC_BASE_URL=https://api.minimaxi.chat/v1/anthropic
  export ANTHROPIC_AUTH_TOKEN=$MINIMAX_API_KEY
  export ANTHROPIC_MODEL=MiniMax-M2.7
  echo "已切换到 MiniMax M2.7（自我进化旗舰）"
}
```

## 🔬 实战案例

### 案例1：Windows 用户 DeepSeek V3.2 从零配置

```powershell
# 第一步：设置环境变量（永久生效）
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL", "https://api.deepseek.com/anthropic", "User")
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_AUTH_TOKEN", "sk-你的DeepSeek_Key", "User")
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_MODEL", "deepseek-chat", "User")
[System.Environment]::SetEnvironmentVariable("API_TIMEOUT_MS", "600000", "User")

# 第二步：重启 Terminal

# 第三步：验证
claude -p "你好，告诉我今天的日期"

# 第四步：测试1M上下文
claude -p "我的文档很长，请帮我总结以下内容：" < 超长文档.txt
```

