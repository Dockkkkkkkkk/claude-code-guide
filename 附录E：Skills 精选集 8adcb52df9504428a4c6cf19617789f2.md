## 开发类 Skills

### 代码审查（[code-review.md](http://code-review.md)）

```markdown
# 代码审查

对当前 Git diff 进行代码审查。

## 安全性
- 是否有 SQL 注入、XSS、CSRF 风险
- 是否有硬编码的 API Key、密码、Token
- 是否有未验证的用户输入

## 代码质量
- 是否有重复代码可以提取为函数
- 是否有明显的性能问题（N+1 查询、内存泄漏）
- 是否符合项目 CLAUDE.md 中的规范

## 输出格式
按以下格式输出 Markdown 报告：
- **必改**：必须修改的问题
- **建议改**：建议优化的地方
- **可选**：可以考虑的改进
```

### Git 提交信息（[git-commit.md](http://git-commit.md)）

```markdown
# 生成 Git Commit 信息

根据当前的 git diff，按照 Conventional Commits 规范生成提交信息。

## 信息格式
<type>(<scope>): <subject>

<body>

<footer>

## 允许的 type
- feat: 新功能
- fix: 修复 Bug
- docs: 文档更新
- style: 代码格式
- refactor: 重构
- test: 测试相关
- chore: 构建/工具

## 要求
- subject 不超过 72 个字符
- 使用中文（除非项目明确使用英文）
- body 说明「为什么」而非「做了什么」

只输出 commit 信息，不要执行 git commit。
```

### 调试助手（[debug.md](http://debug.md)）

```markdown
# 调试分析

分析以下错误并提出修复方案：

## 分析步骤
1. 读取错误信息和堆栈跟踪
2. 定位可能的根本原因（列出 3 个下以候选）
3. 对每个可能原因，证明或证伪
4. 提出最可能的定质以及修复步骤

## 输出格式

**根本原因：** [1-2 句]

**为什么会发生：** [2-3 句]

**修复方案：**
```

具体修改

```

**后续建议：** [如何防止同类问题再次发生]
```

## 内容创作类 Skills

### 广告文案（[ad-copy.md](http://ad-copy.md)）

```markdown
# 广告文案生成

根据提供的产品/服务信息，生成多平台广告文案。

## 生成内容
1. 小红书笔记标题 x5
2. 小红书正文（500-800字）
3. 抖音口播脚本（30秒）
4. 微制标题 x3

## 要求
- 文字要自然流畅，避免广告语气
- 小红书筑记要用第一人称
- 抖音要有鸦头金句
- 超连结合产品中最强的坑点和益处
```

### 日报生成（[daily-report.md](http://daily-report.md)）

```markdown
# 生成工作日报

执行以下步骤生成今日工作日报：

1. `git diff --stat HEAD~1` 获取今日文件变更
2. `git log --oneline --since="today"` 获取今日提交
3. 读取 .claude/worklog.md 中今天的手动备注（如果存在）

整合内容，按以下格式生成日报：

---
【YYYY-MM-DD 工作日报】

**今日完成**
- ...

**遇到的问题**
- ...

**明日计划**
- ...

---

将日报保存到 daily-reports/YYYY-MM-DD.md
```

## 数据分析类 Skills

### CSV 数据分析（[csv-analysis.md](http://csv-analysis.md)）

```markdown
# CSV 数据分析

分析提供的 CSV 文件并输出全面分析报告。

## 分析内容
1. 基本统计：行数、列数、缺失値比例
2. 数据分布：主要数字列的平均値、中位数、最大小値
3. 异常展示：异常値和集中趋势
4. 数据质量评估：缺失关系、格式不一致、重复数据
5. 业务洞察：基于数据特征的欧空分析

输出 Markdown 格式报告，包含具体数字和洞察。
```

## 运维类 Skills

### 部署前检查（[pre-deploy.md](http://pre-deploy.md)）

```markdown
# 部署前安全检查

执行部署前的完整安全检查：

1. 检查硬编码密钥：`grep -r "password\|secret\|api_key\|token" src/ --include="*.ts" --include="*.js"`
2. 运行完整测试套件：`npm test`
3. 检查依赖安全漏洞：`npm audit`
4. 检查 Docker 成像大小：`docker images`
5. 验证数据库迁移文件就绪

**通过条件：** 输出 "READY TO DEPLOY"
**失败条件：** 输出具体问题列表和建议修复方法
```
