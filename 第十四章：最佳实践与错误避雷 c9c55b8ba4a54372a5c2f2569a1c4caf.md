## 14.1 提示词工程学

### 核心原则：越具体越好

```
❓ 模糊提示：「帮我修复 Bug」
✅ 具体提示：「用户执行登录时，如果密码错误的错误次数 ≥ 3 次，
  应返回 429 Too Many Requests，
  目前只返回 200。错误在 auth/login.ts 第 47 行」

❓ 模糊提示：「优化这个项目」
✅ 具体提示：「对 /api/users 接口进行性能优化。
  当前 p95 响应时间 2.3s，目标 < 200ms。
  先分析恒同调用的根本原因，再给出最优方案」
```

### CRIT 框架法

- **C**ontext（上下文）：项目技术栈、相关文件
- **R**equirements（要求）：具体现状和期望结果
- **I**nput（输入）：关键文件或代码片段
- **T**arget（目标）：可验证的完成标准

```
【上下文】
Node.js 20 + PostgreSQL 15，用户设计中心

【要求】
实现用户头像上传功能

【相关文件】
- models/User.ts
- routes/profile.ts

【完成标准】
1. 支持 JPG/PNG/WebP，最大 5MB
2. 自动裁剪为 200x200
3. 存入 /uploads/ 并更新 User.avatar_url
```

## 14.2 Think 模式与思考深度控制

```
# 基础思考（最快，少用 token）
回答我：...

# 标准思考（大多数开发任务的默认）
think about ...

# 深度思考（复杂分析或架构设计）
think hard about ...

# 极限思考（最重要的决策，消耗最多 token）
think harder
ultrathink
```

**使用建议：**

- 日常功能开发 → `think`
- 复杂架构设计 → `think hard`
- 安全和核心算法 → `think harder / ultrathink`

## 14.3 上下文管理最佳实践

**常见坑：上下文窗口填满、响应变度降低**

| 问题 | 自我评估方法 | 解决方案 |
| --- | --- | --- |
| 上下文过长 | 响应变慢、是非增多 | 开新对话或精简 [CLAUDE.md](http://CLAUDE.md) |
| 需要大量文件 | 所有代码都要读 | 先 `ls` 和 `grep`，只加载必要文件 |
| 三天以上的项目 | 上下文大 | 将每个任务帧将开新对话 |

[**CLAUDE.md](http://CLAUDE.md) 自动压缩技巧：**

```bash
# 如果 CLAUDE.md 过长，证 Claude 帮你压缩！
claude "请将 CLAUDE.md 压缩到 500 字以内，
保留最关键的技术展暠和和規范"
```

## 14.4 最常见的 10 个错误

### 错误 1：过度依赖默认设定

```bash
# 错误：完全依赖自动设置，从不查看 settings.json
claude "扫描项目"

# 正确：了解当前配置
claude --settings ~/.claude/settings.json "扫描项目"
# 或先执行 /config 查看当前权限
```

### 错误 2：给整个代码库加载进上下文

```bash
# 错误：一次性把所有文件加入上下文
claude "@./src 我项目有什么问题"

# 正确：让 Claude 自己排查
claude "我的项目操作令行工具被禁用，请先列出目录结构，
再找出关键配置文件并读取内容"
```

### 错误 3：用 [CLAUDE.md](http://CLAUDE.md) 做幸事废万

```markdown
# 错误：过于详细，叠层赘赘
CLAUDE.md > 5000 字的项目：犬 這赀這赀

# 正确：只写必要的内容
- 技术栈：3 行
- 要避免的实践：5 行
- 项目架构要点：5 行
# 总共不超过 500 字
```

### 错误 4：过山过海堆坑

```
# 错误：在同一个对话中要求太多不相关的任务
“修复登录 Bug、重构用户模块、写 README、出单元测试”

# 正确：一个对话一个任务
“修复登录 Bug：密码错误次数过多时应返回 429”
```

### 错误 5：不使用 Auto-Approve，各种确认疑臭

```bash
# 开发者自己的非生产环境：
claude --dangerously-skip-permissions

# 明确允许的工具：
claude --allowedTools "read,write,bash"
```

### 错误 6：忽视 ESC 键

```
一定要知道：
- ESC = 就地取消当前操作（不退出对话）
- ESC + ESC = 跳出 Agent 模式，进入交互模式
展示: 当 Claude 在拼命做一个错误的任务时，及时按 ESC
```

### 错误 7：API 密钥沬露

```bash
# 错误：把密钥直接写入代码
export ANTHROPIC_API_KEY=sk-ant-xxxx  # 加入 git

# 正确：使用 .gitignore 和环境变量
cat >> .gitignore << 'EOF'
.env
.env.local
*.key
EOF

# 使用 .env 文件
echo 'ANTHROPIC_API_KEY=sk-ant-xxxx' > .env
chmod 600 .env
```

### 错误 8：对国产模型配置不起手

```bash
# 错误：只设置 API Key，忘记设置 Base URL
export ANTHROPIC_API_KEY=sk-xxxx  # DeepSeek Key
claude  # 失败！

# 正确：需要同时设置 Base URL
export ANTHROPIC_API_KEY=sk-xxxx
export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
export CLAUDE_MODEL=deepseek-chat
claude  # 成功！
```

### 错误 9：小维修也用 ultrathink

```
# 错误：一个简单任务用了最深度思考
ultrathink

你帮我公司名称首字母转换为大写。
公司名称： hello world startup

# 正确：小任务直接跳过 think
公司名称首字母转大写：hello world startup
```

### 错误 10：完全依赖、不了解 Claude 的能力边界

```
# 能力边界清单：
✅ 擅长：代码生成、重构、文档写作、数据分析、调试已有错误
✅ 擅长：按细节展廿结构化指令执行
✅ 擅长：初始架构和模板

⚠️ 需要人工配合：业务发现和需求定义、关键架构决策、最终上线前复查
```

## 14.5 需求平衡与上下文优化

```bash
# 项目初期：一个对话做开发，上下文少
claude "初始化一个 React + TypeScript 项目"

# 项目中期：每个功能开新对话
claude "实现用户登录模块 @routes/auth.ts"

# 项目后期：专注小任务
claude "修复 这一个具体山调试：@routes/auth.ts:47"
```

### 在对话中确认进度

```
# 不要让 Claude 默默创作——要求先说计划
实现用户登录功能前，先说说你的方案，
我确认后再动手。

# 在复杂任务前，要求小步进行
请先只忾83实现登录接口，不要动前端。
我测试没问题后，再继续分源展冗。
```

## 14.6 不同层次的中文练命令表

```bash
# 初山者最常用
claude                     # 交互对话
/help                      # 查看帮助
/status                    # 查看当前状态
/exit                      # 退出

# 开发者最常用
claude "任务描述"       # 直接开始任务
claude --print "提示"       # 非交互模式
claude --resume              # 恢复上次对话

# 高级用法
claude --allowedTools "read,write,bash"  # 限制权限
claude --model deepseek-chat             # 指定模型
claude --max-turns 20                    # 限制轮数
claude --continue                        # 继续上一次对话
```

