## 11.1 Claude Code GitHub Actions

Anthropic 提供了官方的 GitHub Action：`anthropics/claude-code-action`。

**四大场景：**

1. 通过 `@claude` 指令 PR 审查和修复
2. 就地 航运任务基于 Issues 实现
3. Issue 自动分析与建议
4. 代码质量自动检查

## 11.2 快速添加到你的仓库

### 第一步：配置 Secrets

```bash
# 在仓库 Settings > Secrets > Actions 中添加：

# 必需：
ANTHROPIC_API_KEY=sk-ant-xxxx

# 必需（用于创建 PR、评论）：
GITHUB_TOKEN（自动提供，无需配置）
```

### 第二步：创建 Workflow 文件

**PR 自动审查 + @claude 指令：**

```yaml
# .github/workflows/claude-review.yml
name: Claude Code 审查

on:
  pull_request:
    types: [opened, synchronize]
  issue_comment:
    types: [created]

jobs:
  claude-review:
    if: |
      github.event_name == 'pull_request' ||
      (github.event_name == 'issue_comment' && contains(github.event.comment.body, '@claude'))
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
      issues: write
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: $ secrets.ANTHROPIC_API_KEY 
          github_token: $ secrets.GITHUB_TOKEN 
          # 可选：自定义 Claude 的行为
          claude_instructions: |
            审查代码时重点关注：
            1. 安全漏洞（SQL注入、XSS、硬编码密钥）
            2. 性能问题（N+1查询、内存泄漏）
            3. 是否符合项目的 CLAUDE.md 规范
            给出具体可操作的建议，而非模糊评论。
```

### 第三步：在 PR 中使用

```
# 在 PR 评论中输入：
@claude 请审查这个 PR，重点检查安全漏洞

@claude 这个测试覆盖率有点低，帮我添加几个单元测试

@claude 帮我将这个 PR 进行重构，减少重复代码

# Claude 会直接修改代码并提交新的 commit
```

## 11.3 更多 Workflow 模板

### 自动 Bug 修复（基于 Issue）

```yaml
# .github/workflows/auto-fix.yml
name: 自动修复 Bug

on:
  issues:
    types: [labeled]

jobs:
  auto-fix:
    if: github.event.label.name == 'auto-fix'
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
    steps:
      - uses: actions/checkout@v4
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: $ secrets.ANTHROPIC_API_KEY 
          github_token: $ secrets.GITHUB_TOKEN 
          prompt: |
            请分析并修复以下 Issue:
            标题: $ github.event.issue.title 
            内容: $ github.event.issue.body 
            修复后创建 Pull Request
```

### 每日代码健康检查

```yaml
# .github/workflows/daily-health.yml
name: 每日健康检查

on:
  schedule:
    - cron: '0 9 * * 1-5'  # 周一到周五 09:00

jobs:
  health-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: $ secrets.ANTHROPIC_API_KEY 
          github_token: $ secrets.GITHUB_TOKEN 
          prompt: |
            对整个代码库进行健康检查：
            1. 扫描过期依赖（package.json）
            2. 检查已知的安全漏洞
            3. 识别潜在的拆迁标记（TODO/FIXME/HACK）
            生成 Markdown 报告并作为 Issue 创建
```

### 自动生成 CHANGELOG

```yaml
# .github/workflows/changelog.yml
name: 生成 CHANGELOG

on:
  push:
    tags:
      - 'v*'

jobs:
  generate-changelog:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: $ secrets.ANTHROPIC_API_KEY 
          github_token: $ secrets.GITHUB_TOKEN 
          prompt: |
            基于最近两个标签之间的 Git 提交记录，
            生成用户友好的 CHANGELOG。
            将内容写入 CHANGELOG.md 并创建 PR。
```

## 11.4 对接国产模型 API（CI 坏境配置）

如果在 GitHub Actions 中需要使用国个模型，尝试以下配置：

```yaml
# 在 Secrets 中设置：
BASE_URL=https://api.deepseek.com/anthropic
ANTHROPIC_API_KEY=sk-xxxx  # DeepSeek API Key
CLAUDE_MODEL=deepseek-chat

# 在 Workflow 中使用
en:
  - uses: anthropics/claude-code-action@v1
    with:
      anthropic_api_key: $ secrets.ANTHROPIC_API_KEY 
      github_token: $ secrets.GITHUB_TOKEN 
      base_url: $ secrets.BASE_URL 
      model: $ secrets.CLAUDE_MODEL 
```

## 11.5 防止失控的最佳实践

```yaml
# 建议的安全配置
steps:
  - uses: anthropics/claude-code-action@v1
    with:
      anthropic_api_key: $ secrets.ANTHROPIC_API_KEY 
      github_token: $ secrets.GITHUB_TOKEN 
      # 限制 Claude 的操作权限
      allowed_tools: "read_file,write_file,create_pr_comment"
      # 禁止删除操作
      disallowed_tools: "delete_file,execute_command"
      # 设置超时防止死机
      timeout_minutes: 15
      # 限制 token 消耗
      max_tokens: 10000
```

## 🔬 实战案例

### 案例1：全自动化 PR 审查流程

```bash
# 1. 建立仓库并推送代码
git push origin feature/new-payment-api

# 2. 开启 PR
# GitHub 自动触发 claude-review.yml

# 3. Claude 自动：
#    - 分析 diff
#    - 发现安全问题：API key 硬编码
#    - 在 PR 下方评论具体建议

# 4. 开发者回复："@claude 帮我修复这个安全问题"

# 5. Claude 自动修改代码并提交 commit
```

### 案例2： Issue 标签 auto-fix 自动修复

```
1. 开发者在 GitHub 创建 Issue:
   标题："求证失败时返回 200 而不是 401"
   内容：复现步骤...
   
2. 给 Issue 添加标签: auto-fix

3. GitHub Actions 触发，Claude 自动：
   - 分析 Issue 描述
   - 定位相关代码
   - 修复返回状态码错误
   - 创建 PR 并关联 Issue

4. 开发者只需 Review 和 Merge
```
